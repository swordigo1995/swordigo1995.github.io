Differences between Bump mapping, Normal mapping, Displacement mapping & Parallax mapping

2017-05-03
  • Computer Graphics

Because of the similar function of these three texture mapping method, we often can not distinguish from them clearly.

Now, I’m going to make a summary of the differences between them.

Common aspect:

Same ideas in adding extra details that doesn’t exist in geometry. All a technique that significantly boosts a textured surface’s detail and gives it a sense of depth.

Some comments thinks that Any technique that fakes a relief on flat geometry is called bump mapping. It’s a generic name, and it contains height mapping & normal mapping.
But in this article, we assume the bump mapping is the “height map” and normal mapping is a better method for bump mapping.

Now, we discuss them seperately.

Bump mapping

Input:

The input is height map. Usually a gray scale image.

When values in a bump map are close to 50% gray, there’s little to no detail that comes through on the surface. When values get brighter, working their way to white, details appear to pull out of the surface. To contrast that, when values get darker and closer to black, they appear to be pushing into the surface.

Advantages:

Save space.

Disadvantages:

We need to calculate the normal at that texel based on the height value stored in heightmap, then use the result normal to do shading.

We need to use the height imformation to calculate coresponding normal vector! How to do it?

Normal mapping

A newer, better type of bump map.It makes the output quicker than traditional bump mapping.

Input:

RGB texture

Two different types to present normal maps:
Tangent Space normal maps &　Object Space normal maps (slightly improved performance over tangent map)

Advantages:

Unlike normal calculation in height map(bump mapping), we don’t need to calculate here at all because the normal is already in the normal map texture.

It’s also need to be mentioned that normal map is more easier to integrate into rendering pipeline, especially the support of mobile game engine.

Displacement mapping

Input:

height map similar to bump mapping.

But the usage of them is pretty different. In bump mapping, the height map is used to calculate new normal with derivation from the map, but it is used to simulate the actual deformation of the geometric model in displacement map. In a word, it affects the geometric properties instead of normal vector.

How to infect the actual model?
One way to do this is to take a plane with roughly 1000 vertices and displace each of these vertices based on a value in a texture that tells us the height of the plane at a specific area.

Disadvantages:

A problem with displacing vertices is that a plane needs to consist of a large amount of triangles to get a realistic displacement otherwise the displacement looks too blocky. As each flat surface could then require over 1000 vertices this quickly becomes computationally infeasible.

The significant cost is the main unsatisfactory aspect.

Parallax mapping

A fast approximation method in visual for Displacement mapping.

In general, Parallax mapping is a lot better in conveying a sense of depth and together with normal mapping gives incredibly realistic results.

Input:

The height map same as dispalcement mapping.

Advantages:

The displacement mapping technique that doesn’t require extra vertex data to convey depth. It just makes some changes on the texture color we saw, and this trick obtains a similar result as displacement mapping with little computational cost.

Some tips in modeling technology:

高模与低模的概念与用途。

高模就是分段很细的，有很多很多个面构成，上面有很多节点，这样的模型在操作时运行比较慢，但可以做得很精细；低模相反，但低模再法线贴图或普通凹凸贴图时也能显示的比实际的精细得多，所以很多时候做大场景或特别精细的时候要用高模导出法线贴图，贴到低模上去操作，这样会大大提高效率。
